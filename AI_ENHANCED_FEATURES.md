# 🤖 AI增强功能详解

## 🎯 核心增强

### 1. 智能需求分析
AI现在能够深度理解用户需求，识别：
- **合规要求**：风险管理、审计追踪、监管合规
- **业务重点**：关系网络、时间维度、数据治理
- **图表偏好**：关系箭头显示、布局选择、配色方案

### 2. 增强的关系图表
生成的专业SVG图表包含：
- **实体关系箭头**：Hub→Link→Satellite的连接线
- **箭头类型区分**：实线（Hub→Link）、虚线（Link→Satellite）
- **智能布局**：分层排列，清晰的关系展示
- **配色方案**：专业配色，突出不同实体类型

### 3. 定制化输出
根据用户需求定制：
- **字段映射重点**：业务键、合规字段、关系字段
- **实体扩展**：需要详细展示的实体
- **合规检查**：自动识别和标记合规相关字段

## 📊 图表功能详解

### 实体类型识别
```
🟢 Hub (HUB_Party)       - 业务主键实体
🟡 Link (LNK_Party_Role) - 实体间关系
🟣 Satellite (SAT_Party)  - 描述性属性
🔵 PIT (PIT_Party)       - 时间点视图
🟠 Bridge (BRIDGE_XXX)    - 多对多关系
```

### 关系箭头说明
```
➡️ 实线箭头：Hub → Link (业务关系)
⤍ 虚线箭头：Link → Satellite (属性关系)
```

### 图表布局
```
第1层：Hubs (业务主键实体)
第2层：Links (实体关系)
第3层：Satellites (描述性属性)
第4层：PIT Tables (时间点视图)
第5层：Bridges (复杂关系)
```

## 🧠 AI分析能力

### 支持的需求模式

#### 基础转换
```
"将这个银行业Party数据转换为Data Vault 2.0格式"
→ 识别为标准转换，生成完整的Hub-Link-Satellite结构
```

#### 合规重点
```
"重点关注客户风险管理和合规审计功能"
→ 启用风险管理模式，突出合规字段，添加审计追踪
```

#### 关系网络
```
"创建完整的客户关系网络模型"
→ 启用关系网络模式，强化Link连接，展示关系箭头
```

#### 时间维度
```
"需要支持历史数据追踪和时间点查询"
→ 启用时间维度模式，包含PIT表和历史追踪字段
```

### 智能决策

AI会根据用户输入自动决策：
- **生成哪些实体类型**（Hub/Link/Satellite/PIT/Bridge）
- **突出哪些字段**（业务键/合规字段/关系字段）
- **如何布局图表**（层级/网络/分组排列）
- **是否显示关系箭头**（简单/详细连接）

## 🎨 图表定制选项

### 配色方案
- `professional` - 专业配色（默认）
- `accentuate_compliance` - 突出合规字段

### 布局偏好
- `hierarchical` - 层级布局（默认）
- `networked` - 网络布局
- `grouped` - 分组布局

### 显示选项
- `show_relationship_arrows` - 显示关系箭头
- `detailed_connections` - 详细连接线
- `entity_expansion` - 实体详细展开
